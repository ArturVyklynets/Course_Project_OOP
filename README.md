# Course_Project_OOP
Репозиторій для зберігання інформації про курсовий проєкт - веб-застосунок для оренди житла

# Functional requirements:
 - Як орендар, я хочу мати можливість переглядати основну сторінку з пропозиціями житла.
 - Як орендар, я хочу мати можливість переглядати сторінку з детальною інформацією про конкретний об'єкт житла.
 - Як орендар, я хочу мати можливість пошуку житла за назвою.
 - Як орендар, я хочу мати можливість переглядати історію своїх оренд.
 - Як орендар, я хочу мати можливість фільтрувати житло за рейтингом.
 - Як орендар, я хочу мати можливість фільтрувати житло за вулицею.
 - Як орендар, я хочу мати можливість фільтрувати житло за ціною.
 - Як орендар, я хочу мати можливість залишати відгуки та оцінки для об'єктів житла.
 - Як орендар, я хочу мати можливість подавати скаргу на власника житла.
 - Як орендар, я хочу мати можливість подавати запити на оренду житла.
 - Як власник житла, я хочу мати можливість додавати, редагувати, видаляти житло для оренди.
 - Як власник житла, я хочу мати можливість оцінювати орендаря.
 - Як власник житла, я хочу мати можливість подавати скаргу на орендаря.
 - Як власник житла, я хочу мати можливість переглядати та приймати чи відмовляти запити на оренду житла.
 - Як користувач, я хочу мати можливість авторизовуватися чи реєструватися.
 - Як адміністратор, я хочу мати можливість блокування користувача.
 - Як адміністратор, я хочу мати можливість перегляду, редагування та видаляння оголошення про оренду житла.
 - Як адміністратор, я хочу мати можливість перегляду та видалення відгуків користувачів.

# System behaviours: 
1. Авторизація та аутентифікація користувачів:
   - Реєстрація користувача (User Registration)
     - Вхідні дані: логін, ім'я, електронна пошта, пароль.
     - Процес: користувач заповнює форму реєстрації. Система перевіряє унікальність електронної пошти та зберігає новий запис у базі даних.
     - Вихідні дані: підтвердження реєстрації.
       
   - Аутентифікація користувача (User Authentication)
     - Вхідні дані: логін, пароль.
     - Процес: користувач вводить свої дані для входу. Система перевіряє відповідність даних у базі.
     - Вихідні дані: доступ до системи або повідомлення про помилку.
 
2. Функціональність для орендарів:
   - Перегляд детальної інформації про конкретний об'єкт житла.
     - Вхідні дані: ідентифікатор об'єкта житла, який користувач хоче переглянути.
     - Процес: користувач обирає конкретний об'єкт житла для перегляду детальної інформації. Система отримує ідентифікатор об'єкта житла. За допомогою ідентифікатора система знаходить в базі даних відповідний об'єкт житла.
     - Вихідні дані: якщо об'єкт знайдено, система повертає детальну інформацію про нього.Якщо об'єкт не знайдено, система повідомляє про помилку.
       
   - Фільтрація житла за рейтингом.
     - Вхідні дані: рейтинг, за яким користувач хоче здійснити фільтрацію.
     - Процес: користувач встановлює параметри фільтрації, вказуючи мінімальний рейтинг, який він хоче побачити у житлі. Система отримує від користувача обрані параметри фільтрації. Система перевіряє рейтинг кожного об'єкта житла в базі даних.
     - Вихідні дані: список об'єктів житла, які мають рейтинг не нижче встановленого користувачем, що відображаються на сторінці.
       
   - Подання запитів на оренду житла.
     - Вхідні дані: інформація про користувача, який подає запит на оренду, та об'єкт житла, на який він подає запит. Ця інформація може включати ім'я користувача, його контактні дані (наприклад, електронну пошту або номер телефону), а також додаткові дані про об'єкт житла, такі як його ідентифікатор або адреса.
     - Процес: користувач обирає конкретний об'єкт житла, на який він бажає подати запит на оренду. Користувач заповнює форму з інформацією про себе та свої контактні дані. Система отримує дані про користувача та об'єкт житла, на який він подає запит. Запит на оренду додається до бази даних, де він може бути переглянутий власником об'єкта житла. Якщо запит успішно додано до бази даних, система надсилає підтвердження користувачеві.
     - Вихідні дані: підтвердження користувачеві про успішне подання запиту на оренду житла. Власник об'єкта житла може також отримати повідомлення або можливість переглянути всі запити на оренду для свого об'єкта.
       
   - Перегляд основної сторінки з пропозиціями житла.
   - Пошук житла за назвою, вулицею та ціною.
   - Перегляд історії своїх оренд.
   - Залишання відгуків та оцінок для об'єктів житла.
   - Подання скарг на власників житла. 
     
4. Функціональність для власників житла:
   - Додавання, редагування та видалення житла для оренди.
     - Вхідні дані: інформація про житло, яке користувач бажає додати, відредагувати або видалити. Ця інформація може включати такі дані, як назва об'єкта житла, адреса, опис, ціна за ніч, зображення та будь-які інші характеристики житла.
     - Процес:
        - Додавання житла для оренди:
          - Користувач заповнює форму з інформацією про нове житло для оренди.
          - Система отримує ці дані і додає новий об'єкт житла до бази даних.
        - Редагування існуючого житла для оренди:
          - Користувач вибирає об'єкт житла, який він бажає відредагувати, та вносить зміни до його характеристик.
          - Система оновлює відповідний запис про об'єкт житла в базі даних.
        - Видалення житла для оренди:
          - Користувач вибирає об'єкт житла, який він бажає видалити.
          - Система видаляє відповідний запис про об'єкт житла з бази даних.
     - Вихідні дані: підтвердження користувачеві про успішне додавання, редагування або видалення об'єкта житла для оренди. В разі успішної операції користувач може отримати підтвердження через електронну пошту або сповіщення на сторінці.
       
   - Оцінювання орендарів.
     - Вхідні дані: ідентифікатор орендаря, якого користувач бажає оцінити, та його оцінка.
     - Процес: користувач обирає орендаря, якого він бажає оцінити. Користувач вказує оцінку, яку він хоче поставити орендарю. Система отримує введені дані та оновлює рейтинг оцінки орендаря в базі даних.
     - Вихідні дані: Підтвердження користувачеві про успішне оцінювання орендаря. У разі успішного оновлення рейтингу користувач може отримати підтвердження через електронну пошту або сповіщення на сторінці.
       
   - Подання скарг на орендарів.
     - Вхідні дані: ідентифікатор орендаря, на якого користувач бажає подати скаргу, та опис скарги.
     - Процес: користувач обирає орендаря, проти якого він має скаргу, та надає деталі скарги. Користувач вказує причину своєї скарги та надає всі необхідні деталі. Система отримує інформацію про скаргу та зберігає її в базі даних для подальшого розгляду.
     - Вихідні дані: підтвердження користувачеві про успішне подання скарги на орендаря. У разі успішного збереження скарги користувач може отримати підтвердження через електронну пошту або сповіщення на сторінці.
       
   - Перегляд та прийняття чи відмова від запитів на оренду житла.
     - Вхідні дані: ідентифікатор запиту на оренду житла та рішення власника житла щодо цього запиту (прийняття або відмова).
     - Процес: власник житла переглядає отримані запити на оренду житла. Для кожного запиту власник житла розглядає інформацію про орендаря та об'єкт житла, на який подано запит. Власник житла приймає рішення щодо кожного запиту, вибираючи прийняття або відмову. Система оновлює статус кожного запиту в базі даних відповідно до прийнятого рішення.
     - Вихідні дані: підтвердження власникові житла про успішне прийняття або відмову від запиту на оренду житла. У разі успішного оновлення статусу запиту користувач може отримати підтвердження через електронну пошту або сповіщення на сторінці.
     
5. Функціональність для адміністратора:
   - Блокування користувачів.
     - Вхідні дані: ідентифікатор користувача, якого адміністратор бажає заблокувати, та причина блокування (опціонально).
     - Процес: адміністратор обирає користувача, якого він бажає заблокувати. Адміністратор вказує причину блокування (якщо це потрібно) та надає всі необхідні деталі. Система оновлює статус користувача в базі даних, встановлюючи його як заблокованого.
     - Вихідні дані: підтвердження адміністраторові про успішне блокування користувача. У разі успішного оновлення статусу користувача адміністратор може отримати підтвердження через електронну пошту або сповіщення на сторінці.

   - Перегляд, редагування та видалення оголошень про оренду житла.
     - Вхідні дані: ідентифікатор оголошення про оренду житла, яке адміністратор бажає переглянути, відредагувати або видалити.
     - Процес: адміністратор переглядає список оголошень про оренду житла. Для кожного оголошення адміністратор може виконати одну з наступних дій:
       - Перегляд: Детальний огляд інформації про об'єкт житла, оцінок, відгуків та іншої важливої інформації.
       - Редагування: Внесення змін у характеристики оголошення, такі як адреса, ціна, опис тощо.
       - Видалення: Прийняття рішення про видалення оголошення з системи.
     - Вихідні дані: підтвердження адміністраторові про успішне виконання дії (перегляд, редагування або видалення) для обраного оголошення про оренду житла. У разі успішної операції адміністратор може отримати підтвердження через електронну пошту або сповіщення на сторінці.

   - Перегляд та видалення відгуків користувачів.
     - Вхідні дані: ідентифікатор відгуку користувача, який адміністратор бажає переглянути або видалити.
     - Процес: адміністратор переглядає список відгуків користувачів. Для кожного відгуку адміністратор може виконати одну з наступних дій:
       - Перегляд: Огляд інформації про відгук, такої як автор, дата, текст тощо.
       - Видалення: Прийняття рішення про видалення відгуку з системи.
     - Вихідні дані: Підтвердження адміністраторові про успішне видалення відгуку користувача. У разі успішної операції адміністратор може отримати підтвердження через електронну пошту або сповіщення на сторінці.
     
# REST API endpoints:
1. Авторизація та аутентифікація користувачів:
   - POST /api/auth/login - Авторизація користувача з використанням логіна та пароля.
   - POST /api/auth/register - Реєстрація нового користувача.
   
2. Функціональність для орендарів:
   - GET /api/properties - Отримання списку пропозицій житла (основна сторінка).
   - GET /api/properties/{property_id} - Отримання детальної інформації про конкретний об'єкт житла.
   - GET /api/properties/search?query={query} - Пошук житла за назвою.
   - GET /api/properties/filter?street={street}&price={price}&rating={rating} - Фільтрація житла за вулицею, ціною та рейтингом.
   - GET /api/rent_history - Отримання історії оренд.
   - POST /api/reviews/{property_id} - Додавання відгуку та оцінки для об'єкта житла.
   - POST /api/complaints/{owner_id} - Подання скарги на власника житла.
   - POST /api/rent_requests - Подання запиту на оренду житла.
     
4. Функціональність для власників житла:
   - POST /api/properties - Додавання нового об'єкта житла.
   - PUT /api/properties/{property_id} - Редагування інформації про об'єкт житла.
   - DELETE /api/properties/{property_id} - Видалення об'єкта житла.
   - POST /api/property_reviews/{review_id}/rate - Оцінювання орендаря.
   - POST /api/tenant_complaints/{tenant_id} - Подання скарги на орендаря.
   - GET /api/rent_requests - Отримання списку запитів на оренду житла.
   - PUT /api/rent_requests/{request_id}/accept - Прийняття запиту на оренду житла.
   - PUT /api/rent_requests/{request_id}/reject - Відмова від запиту на оренду житла.

6. Функціональність для адміністратора:
   - POST /api/admin/block_user/{user_id} - Блокування користувача.
   - GET /api/admin/properties - Отримання списку оголошень про оренду житла для перегляду, редагування та видалення.
   - DELETE /api/admin/properties/{property_id} - Видалення оголошення про оренду житла.
   - GET /api/admin/reviews - Отримання списку відгуків користувачів для перегляду та видалення.
   - DELETE /api/admin/reviews/{review_id} - Видалення відгуку користувача.
